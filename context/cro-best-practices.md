# CRO Best Practices for TheHues Landing Pages

Conversion guidelines for TheHues (English site, US-focused users), especially for collection and custom-product journeys.

---

## Primary Conversion Goals

1. Start product customization on collection/PDP pages
2. Add configured product to cart
3. Use pre-purchase confidence tools (swatches, measurement guides, visualization, free design service)
4. Complete checkout

---

## Above-the-Fold Rules (Collection Pages)

### 5-Second Clarity Check
Visitors should immediately understand:
1. What category this page solves (e.g., blackout, thermal, soundproof, custom)
2. Why this collection matters (performance + style)
3. What to do next (browse SKUs or start customization)

### Required Elements
| Element | Purpose | TheHues Guidance |
|---|---|---|
| H1 + subvalue | Relevance + clarity | Use function + benefit language (e.g., sleep/privacy/insulation/noise) |
| Product grid immediately visible | Fast exploration | Prioritize best sellers and function-first tags |
| Trust/certainty strip | Reduce hesitation | Mention custom-fit guidance, support availability, or tool links |
| Primary CTA path | Drive action | "Customize", "Order Swatches", or "Use Measurement Guide" |

### Above-the-Fold Don’ts
- Don’t force users to scroll past long intros before product cards.
- Don’t show multiple conflicting primary CTAs.
- Don’t hide support links for measurement and style choice.

---

## Product Page CRO (Custom Configuration)

### Critical Components
1. **Configuration clarity**: Size, header, lining, panel count should be sequential and understandable.
2. **Risk reduction nearby**: Link to measurement guide and policy content near configuration inputs.
3. **Decision aids in-line**: Surface visualization/free design options before Add to Cart.
4. **Microcopy on complex options**: Explain memory shape training, tiebacks, and specialty add-ons in plain language.

### Configuration UX Rules
- Keep one clear step order: `Measure -> Configure -> Preview -> Add to Cart`.
- Use short helper text under each complex option.
- Auto-highlight required vs optional choices.
- Show estimated arrival timing only from verified policy language.

## CTA Strategy

### By Funnel Stage
| Stage | Recommended CTA | Avoid |
|---|---|---|
| Early research | "Use Measurement Guide" | "Buy Now" too early |
| Mid consideration | "Try Visualization Tool" / "Get Free Design Mock-up" | Generic "Learn More" |
| Ready to buy | "Customize & Add to Cart" | Unclear or multiple equal-priority CTAs |

### CTA Copy Principles
- Verb + object + benefit
- Example: "Customize Blackout Curtains for Your Window"
- Keep language specific to collection intent

---

## Trust & Risk Reversal

### Highest Impact Signals for TheHues
1. Clear custom-order process guidance
2. Measurement resources before purchase
3. Policy transparency (modification windows, quality-issue process)
4. Trade/project support where relevant

### Placement
- Near configuration blocks
- Near Add to Cart button
- Near delivery expectation area

### Risk Reversal Copy Blocks
- "Not sure on size? Check the measurement guide first."
- "Need visual confirmation? Use our visualization tool or free design service."
- "Review custom-order policy details before checkout."

---

## Objection Handling Matrix

### "I’m worried custom size will be wrong"
- Put measurement guide links beside size input.
- Add contextual examples for panel count and fullness.

### "I can’t picture this in my room"
- Promote visualization tool and free design service before checkout.

### "I don’t know if blackout/thermal/soundproof is right"
- Add concise comparison blocks on collection pages.
- Link to intent-matched educational posts.

### "What if I need to change details later?"
- Link refund-policy sections on cancellation/modification rules near checkout start.

---

## Internal Linking for CRO + SEO

Each high-intent collection page should link to:
1. One relevant measurement/resource page
2. One confidence tool page (visualization or free design service)
3. 2-3 intent-matched blog guides

Recommended anchors:
- "how to measure curtains"
- "try the visualization tool"
- "compare blackout vs thermal curtains"

---

## Mobile Conversion Rules

- Keep sticky CTA visible after key option selections.
- Collapse long explanatory text into accordions.
- Ensure tap targets are comfortable for option selectors.
- Keep policy/support links visible without forcing desktop-only layouts.

---

## A/B Testing Priorities (TheHues-Specific)

### High Impact
1. Collection hero value proposition language by function
2. Primary CTA wording on collection pages
3. Order and grouping of configuration options
4. Placement of measurement/design-support links near CTA
5. Product-card metadata (functional tags, quick-view content)

### Medium Impact
6. Trust signal format near Add to Cart
7. Comparison table format for blackout/thermal/soundproof
8. Microcopy around optional add-ons
9. Default panel-count recommendation UI

### Low Impact
10. Button style variations
11. Minor iconography changes
12. Footer-level copy tweaks

## CRO Checklist

### Collection Page
- [ ] Function and benefit are clear above fold
- [ ] Product cards visible without long intro scroll
- [ ] Primary CTA path is singular and obvious
- [ ] Links to measurement/design tools are visible

### Product Page
- [ ] Configuration sequence is easy to follow
- [ ] Required options are clearly labeled
- [ ] Policy/support links shown near decision points
- [ ] Add to Cart is clear and unobstructed

### Trust & Support
- [ ] Support paths are easy to find
- [ ] Risk-reversal copy is near CTAs
- [ ] No ambiguous claims that exceed policy text

### Mobile
- [ ] Key actions visible without excessive scroll
- [ ] Option selectors are mobile-friendly
- [ ] Long text is collapsible/scannable

---

**Note**: Keep all conversion claims aligned with current site policy and collection-page copy. Avoid unverified promises.
